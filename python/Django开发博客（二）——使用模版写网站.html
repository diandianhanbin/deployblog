                # 前提

                我们应该把精力放在写Python上，而不是写前端的样式，所以我比较喜欢的就是拿一个模版来改一改，就可以获得一个漂亮的界面，否则，你需要好好的了解了解html和css相关的知识。

                # 版本相关

                操作系统：Mac OS X EI Caption

                Python版本：3.4

                Django版本：1.9

                IDE：PyCharm

                # 模版下载

                网络上应该有很多很多的模版可以下载，可以直接找Django的源码下载，不过我是不太推荐这样，因为下下来的东西你不知道要如何实现它。对于学习Django来说没有什么好处，本次演示的模版下载地址：黑色质感时间轴html5个人博客模板我个人比较喜欢这种黑色系的风格。

                # 数据库系统

                数据库目前我弄了两个简单的表来处理数据

                About Me

                网名：Sven Weng | 点点寒彬

                职业：测试工程师

                公司：XXX

                Email：XXX

                这部分在model里面，用一个表来处理这部分数据，记录这个主要是为了以后拓展使用

                # 视图函数

                视图函数中我们需要返回给index.html两个数据库的信息，一个是UserInfo中的我的个人信息，另外一个是文章的信息，代码如下：

                views.py

from django.shortcuts import render
from .models import UserInfo, BlogBody


def index(request):
    userinfo = UserInfo.objects.first()
    blog_body = BlogBody.objects.all()[:6]
    return render(request,
                  'index.html',
                  {'userinfo': userinfo, 'blog_body': blog_body})

                # 地址配置

                我们需要给视图函数配置一个地址。按照《Django1.9快速写简单的博客》把地址链接到自己应用的urls。

                app/urls.py

from django.conf.urls import url
from . import views

urlpatterns = [
    url(r'^$', views.index, name='index'),

                # 模版修改

                模版文件中有index.html list.html view.html这些页面使用的都是同一个头部和脚部。所以把这部分内容单独抓出来写一个base.html文件，所有页面都继承这个就行了。在头部和脚部中间加上

                {% block bodycontent %}

                # 思路

                首先，首页中多篇文章的显示方式是相同的，也就是说我们可以使用一个for循环来生成多个数据。其次，我们需要把每一篇文章的数据用数据库的内容来填充，最后，点击标题后可以跳转到文章详细内容（后续再谈）
