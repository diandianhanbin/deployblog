                    测试五年——我的2016年度总结

                    sven_weng 2017-01-17 18:01:46 个人总结

                    # 概述

                    回看2015年度总结，Flag年常是完不成的。。。。

                    回想自己的2016年，还是比较充实的，总体来说，干了下面几件事：

                    • 读了一些书
                    • 了解了好多乱七八糟的语言
                    • 写了好多篇文章
                    • 瘦了好多
                    • 还有一件低调的事

                    广告：我写的文章都收录在http://wybblog.applinzi.com/blog/

                    # 读书

                    说起读书，应该是我个人比较喜欢的事，今年看了这么些书：

                    • 《向上管理》
                    • 《南朝，北朝(全新升级版) (易中天中华史 12)》
                    • 《魏晋风度(全新升级版) (易中天中华史 11)》
                    • 《这些人，那些事》
                    • 《三体全集》
                    • 《三国史话 (国民阅读经典)》
                    • 《精进：如何成为一个很厉害的人》
                    • 《Wireshark数据包分析实战(第2版)》
                    • 《精通移动App测试实战:技术、工具和案例》
                    • 《中国通史 (时间的轨迹-不可遗忘的历史系列)》
                    • 《零成本实现Android/iOS自动化测试:基于Appium和Test Perfect》
                    • 《MacTalk·人生元编程》
                    • 《编写高质量代码：改善Python程序的91个建议 (Effective系列丛书)》
                    • 《图解HTTP》
                    • 《腾讯Android自动化测试实战》
                    • 《移动App性能评测与优化》
                    • 《大话APP测试2》
                    • 《Selenium2自动化测试实战》
                    • 《Python接口开发与测试》
                    • 《docker_从入门到实践》
                    • 《如何变得有思想?》
                    • 《邓小平传》

                    去年定下来要看的书，结果就读完了一本《图解HTTP》。除了为了糊口去看测试相关的书，历史书还是我最喜欢的类型。读万卷书，行万里路，人生苦短，没办法经历太多，那么就从书中了解一些不一样的思想。而且静下心来看一本书，喝一壶茶，在喧嚣快节奏的城市中，也不失为一种宁静的生活方式。

                    写总结的时候列出的书单，吓了自己一跳，我自己都没感觉自己看了这么多书。

                    # 学习

                    今年算是努力学习的一年，先后了解了Python、HTML、CSS、JS、Django、jQuery、Ajax、HTTP、Android、OC、Java等开发相关的知识，当然，算得上很熟悉的，也就只有Python，其他都是简单的了解，大概知道有那么回事。

                    今年工作的重心都在移动端上，突然发现打开了一扇新的大门，移动端的测试和之前服务端的测试区别太大了。比如通常说的性能测试，指的是高并发下程序的处理逻辑，比如高IO等问题。但是移动端的性能却是页面绘制、页面渲染、CPU、内存，流量、耗电量等方向，和以前的内容完全不同。年初的时候针对这些方面进行过深度的学习和了解，并且写了一个自己的平台。当然，当时的代码功底很烂，写出来之后后续也没有进行维护，就一直烂尾在那了。

                    自动化的方向也完全不一样，服务端的自动化主要实现在接口层，HTTP或者Web Service。而移动端则是UI的自动化，相应的测试框架有非常多。而且现阶段的APP已经基本是原生+H5来实现，UI的自动化还要针对这种混合应用。在Testerhome社区学习了一段时间后，抛弃了原有的关键字的想法，自己封装了一套Appium的测试框架，基本能够满足大部分Android（包括混合APP）应用的UI自动化。

                    今年还学习了一些测试常用的工具，比如抓包工具Wireshark、Charles、Docker、Jenkins，在涉及到网络相关功能的测试时抓包工具能够帮我快速的定位问题。当然，也使用它们成功定位了生产的问题。构建某些环境时Docker简直就是神器。Jenkins今年被我用来自动打包APP、配合Jmeter和Ant定时执行接口测试。

                    # 写文章

                    今年是学习的一年，在学习的过程中我记录了很多学习过程中遇到的问题和相应的解决方案、学习的过程记录、还有一些自己的思想记录。算上这篇总结刚好一百篇，字数超过18W，虽然文笔并不好，但是算是给自己的学习一个交代，看了一下别人的留言，还算对一些人有帮助，也勉强算为这个小圈子做一点贡献吧。

                    当然，在与读者互动的时候，慢慢的也发现了一些问题，大部分的人还是愿意做伸手党，很多问题一定是要自己才能够成为自己的知识。发现问题自己不思考，不动手查，不尝试，就指望别人给你现成的答案，那样的学习其实是没意义的。

                    # 测试

                    2016年是干测试的第五个年头了，感觉每年总结想说说测试的时候，都会有不同的想法，比如今年又有一些以前没有的想法。

                    去年的总结中我提到了，测试应该是一个多面手，各方面都要会一些，今年我觉得还不够。

                    • 测试左移

                    这个思想是去年底萌发的，不过并没有去深度思考，也不会命名一个这么高端的名字，原文点这里.这种方式在大团队其实是比较少的，反而在小团队表现的非常明显。

                    上图（来源：TMQ）是一个流程图例，需求返工导致加剧开发测试成本。

                    一、在需求开始阶段就需要了解需求，对需求做一个正确的判断，而不是提测之后盲目的按照需求文档的规定来测试，测试代码正确性的同时，需要思考需求是否合理。

                    二、测试作为产品的第一个用户，应该多思考。需求是否正确的同时，还应该思考需求是否有价值，如果一个需求仅仅是为了实现一个无用的功能，那么在需求评估阶段，就应该提出来，避免无意义的开发和测试。

                    三、CR也是非常有意义的“左移”事项，不用非常深厚的代码功底，也不需要理清开发的实现框架，只要了解业务逻辑对应的控制代码，可以在用例阶段砍掉很多不必要的测试用例，减少工作量的同时又能保证代码的质量，还能从开发的角度去理解业务的实现。

                    • 测试应该能够提供解决方案

                    这点我觉得是今年全新领悟的点，在这之前，我一直都认为能做到测试左移，就是一个好测试了。但是今年工作上的一些事情，让我觉得，仅仅做到熟悉需求，了解程序逻辑和定位问题是不够的，一个好的测试，还应该能够提供解决方案。

                    之前的测试左移已经阐述了，测试人员应该深度的了解需求，评估需求。那么测试人员就应该对需求有一个自己的认识。同时，测试人员又尽可能的了解程序的控制逻辑。所以测试人员应该比开发更熟悉需求，比产品更了解代码，站在这个高度上，一旦出现了某些问题，测试人员应该要能做到快速定位问题并且根据自己对需求和程序的理解，提供一些解决方案。这种情况在小公司小团队表现的尤为明显。

                    # 减肥

                    恩，这个东西现在我非常有发言权，毕竟一个多月瘦了20斤，还是一件比较了不起的事。

                    减肥肯定不是不吃饭，该吃的要吃，不吃饭哪有力气干活？早饭中饭我都是正常的吃，该怎么吃怎么吃，晚饭尽量少吃，以前我都是能吃多少吃多少。现在尽量控制一下，吃个七八分饱就行了。零食这种东西肯定是不能吃的。如果肚子会饿，吃一些低糖的水果或者黄瓜什么的稍微填一下肚子。精米改成粗粮，比较不容易饿。

                    运动肯定还是要的，每天躺着睡觉肯定不行。适当的跑跑步，走走路，打打球什么的，也有利于减肥。

                    减肥是一项坚持的事情，如果没办法坚持，还是不要轻易减肥，毕竟反弹之后会比原来更可怕。

                    # 遗憾

                    遗憾当然也还是有的。

                    • 没有写出iOS和Android的博客客户端

                    在写博客的时候，我只写了一个Web端，移动端还没有写完。今年的目标其实是在年底的时候把博客的Android和iOS端写出来。不过现在看来是完不成了，这个遗憾留到明年吧。

                    • 没能写出一篇关于区块链的文章

                    区块链这个话题其实是我一直很想写的，比如著名的比特币就是基于区块链技术产生的产品。但是区块链涉及面太广了，先后写了四篇，但是都不满意，全部删了。区块链作为新兴的技术，应该是大有可为的，有兴趣的朋友可以看看《blockchain_guide》这本书，在Github上开源了。

                    • 2015年关于读书的Flag没有完成

                    2015年留给自己的书单并没有读完，有好几本是读了一些，希望2017年能完成它们。

                    # 五年

                    不知不觉就工作五个年头了，时光飞逝，刚来深圳工作时，压根就没想过会发展成什么样，只想着自己能够独立，不再依赖家里。回首这五年，突然觉得成长了好多，感谢一路陪伴成长的亲人，同事、朋友和一些不认识但指导过我的网友。

                    下一个五年过完，就工作十年了，想想都可怕，第一个五年已经从“哥哥”变成“叔叔”了，不知道下一个五年会变成啥？
